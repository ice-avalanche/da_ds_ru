# Урок 1. Введение в Yandex DataLens

## 1. Назначение инструмента

Yandex DataLens – BI-платформа для анализа и визуализации данных, которая используется для построения управленческой и финансовой отчетности, мониторинга KPI и анализа показателей компании.

Позволяет:
- подключаться к различным источникам данных;
- подготавливать и объединять данные;
- строить интерактивные визуализации;
- собирать дашборды;
- публиковать отчеты для совместной работы.

## 2. Логика работы в DataLens

Рабочий процесс строится по схеме: Источник данных → Датасет → Чарт → Дашборд.

### 2.1 Источник данных
Подключение к данным:
- Excel
- CSV
- Google Sheets
- Базы данных

### 2.2 Датасет
Подготовка данных:
- изменение типов полей;
- создание вычисляемых показателей;
- настройка агрегаций;
- объединение таблиц.

### 2.3 Чарт
Построение визуализации:
- выбор измерений и показателей;
- настройка агрегации;
- сортировка;
- фильтрация;
- форматирование.

### 2.4 Дашборд
Сборка нескольких чартов:
- размещение графиков;
- добавление фильтров;
- настройка интерактивности;
- публикация.

## 3. Интерфейс системы

Основные сущности:

| Раздел | Назначение |
|--------|------------|
| Источники | Подключение данных |
| Датасеты | Подготовка структуры |
| Чарты | Отдельные визуализации |
| Дашборды | Комплексные панели |
| Фильтры | Управление отображением |

## 4. Типы данных и роли полей

### 4.1 Типы полей

| Тип | Назначение |
|------|------------|
| Dimension (Измерение) | Категории (регион, продукт, дата) |
| Measure (Показатель) | Числовые значения (выручка, количество) |
| Дата | Временные поля |
| Текст | Строковые значения |

### 4.2 Основные агрегации

| Агрегация | Назначение |
|------------|------------|
| SUM | Сумма |
| AVG | Среднее |
| COUNT | Количество |
| COUNTD | Уникальные значения |
| MAX / MIN | Максимум / минимум |

## 5. Основные типы визуализаций

| Тип | Применение |
|------|------------|
| Столбчатая диаграмма | Сравнение категорий |
| Линейный график | Динамика во времени |
| Круговая диаграмма | Структура |
| Таблица | Детализация |
| KPI | Ключевой показатель |
| Комбинированная | Несколько метрик |

## 6. Фильтрация и интерактивность

Фильтры могут быть:
- по значению;
- по диапазону;
- по дате;
- множественного выбора.

Фильтр может применяться:
- к одному чарту;
- ко всему дашборду.

## 7. Практическая часть

### Задача:
Построить финансовый дашборд, включающий:

- выручку по регионам;
- динамику по месяцам;
- структуру продаж;
- KPI по общей выручке.

### Этапы выполнения:
1. Загрузка Excel-файла.
2. Проверка типов данных.
3. Создание датасета.
4. Построение 3–4 чартов.
5. Сборка дашборда.
6. Добавление фильтра по дате.
7. Публикация.

## 8. Типичные ошибки

| Ошибка | Причина |
|---------|----------|
| Неверная агрегация | COUNT вместо SUM |
| Дата определяется как текст | Ошибка формата |
| Дублирование данных | Ошибка объединения |
| Некорректные итоги | Фильтр не применен |
